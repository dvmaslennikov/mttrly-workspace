# Daily Process (SOP)

## 0) Режим
- Default: draft-first (ничего не публикуем автоматически)
- Публикация только после явного аппрува Димы

## 1) Два прохода в день (CET)
- Утро: 09:30–10:00
- Вечер: 18:00–18:30

## 2) Сбор кандидатов
1. Прогнать запросы из `query-fanout.md`
2. Собрать 20–40 свежих твитов (желательно < 24ч)
3. Удалить мусор: web3-шум, оффтоп, не-ICP

## 3) Скоринг
- Применить правила из `scoring-and-routing.md`
- Отобрать Top-10
- Из Top-10 выбрать Top-5 для reply-пака

## 4) Подготовка reply-пака
Для каждого из Top-5:
- Ссылка на твит
- Почему релевантно (1 строка)
- 2 варианта ответа:
  - SAFE (нейтрально-экспертный)
  - PUNCHY (чуть острее, но без токсика)
- Метка линка: `NO_LINK` / `SOFT_LINK` / `LINK_OK`

## 5) Quality gate перед отправкой
Проверить каждый ответ:
- Полезен сам по себе без перехода на сайт
- Не выглядит как реклама
- Короткий (обычно 1–3 предложения)
- Нет неподтверждённых цифр/кейсов
- Не конфликтует с `response-policy.md`

## 6) Аппрув
- Отправить Диме daily pack
- Ждать явного: какие ответы публиковать

## 7) Логирование
Сохранять в `daily-template.md`-формате:
- Что искали
- Что выбрали
- Что отправили в аппрув
- Что было опубликовано
- Наблюдения (что зашло/не зашло)
